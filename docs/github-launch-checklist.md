# GitHub Launch Checklist

Use this checklist after pushing the repository to GitHub.

## Repository Settings

- Set **Description**:
  - `Local-first token usage analytics for Codex, Claude, Gemini, Cursor, and other AI coding tools.`
- Set **Website** if a demo URL exists.
- Add Topics:
  - `token-usage`
  - `llm`
  - `ai-tooling`
  - `developer-tools`
  - `analytics`
  - `dashboard`
  - `local-first`
  - `rust`
  - `react`
  - `sqlite`

## Social Preview

- Upload `docs/assets/token-insight-preview.png` as the base visual draft.
- Recommended final social preview export:
  - PNG
  - 1280x640
  - less than 1 MB
  - short English headline

## Community Profile

- Keep `README.md` and `CONTRIBUTING.md` up to date.
- Add `CODE_OF_CONDUCT.md` when opening the project to broad contributions.
- Add issue/PR templates when triage traffic grows.

## Pre-Announcement Sanity

- Ensure quick start commands in README run on a clean machine.
- Ensure screenshots match current UI language behavior.
- Verify no local secrets or personal paths are leaked in docs or fixtures.
